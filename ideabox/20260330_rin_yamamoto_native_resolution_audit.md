**Idea:** Audit all remaining ENV and style frame generators still drawing at 1920×1080 then using thumbnail() to downscale to 1280×720. The LANCZOS downscale anti-aliases thin outline pixels (1px–2px) with their neighboring fill colors, creating blended pixels that pollute the render_qa LAB ΔE color fidelity check for any canonical color used as an outline at that weight. The fix (native 1280×720 canvas) is a one-line change per file (W, H = 1280, 720), since all geometry is expressed as fractions of W/H. This C41 cycle found UV_PURPLE ΔE of 27.78 in SF03 and 27.37 in the ENV — entirely caused by this pattern. A batch audit script could grep for "W, H = 1920, 1080" + "thumbnail" in all LTG_TOOL_bg_*.py and LTG_TOOL_style_frame_*.py generators and flag them for conversion. Kai Nakamura could run this as a static-analysis CI check that warns on any generator using 1920×1080 as its draw canvas.
**Benefits:** All ENV and style frame generators affected; fixes a systematic source of false render_qa color fidelity failures that has caused multi-cycle backlogs. Resolves measurement noise in LAB ΔE checks and makes color compliance reporting trustworthy for all thin-outline colors in all scenes.
