**Idea:** Extend LTG_TOOL_palette_warmth_lint_v002 to support a "strict warm" vs "soft warm" mode. Currently the tool flags any entry where G > R OR B > R. But some warm-adjacent colours (like dusty lavenders or warm grays) may intentionally have B slightly above R while still reading warm visually. A "soft warm" mode could use a configurable tolerance per channel (e.g. B may exceed R by up to 10 counts without flagging). This would allow the tool to gate-check truly cold entries (like the old CHAR-M-11 Deep Sage, G=122 >> R=90) while not flagging borderline warm-neutral entries. The tolerance would be a second JSON config key: `"soft_tolerance": {"G": 0, "B": 10}` (0 = strict, 10 = soft). Default stays at 0 (strict) to preserve current behaviour.
**Benefits:** Reduces false positive friction as the warm-character palette grows. Currently only CHAR-M has 11 entries; if we expand to other warm characters (e.g. Luma's grandmother-adjacent flashback characters), the strict mode may flag deliberate warm-neutral bridge colours. Soft mode would still catch the real errors.
