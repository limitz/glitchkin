**Idea:** Build `LTG_TOOL_luma_face_render.py` — a helper wrapper around the new `LTG_TOOL_luma_face_curves.py` that generates a standalone 600×600px face PNG for any expression on demand. Primarily useful for Maya Santos (expression sheet integration) and Jordan Reed (style frames: draw_luma_face_sprint currently uses its own ad-hoc face drawing which would benefit from the new bezier system). The wrapper would accept `--expression NAME --output PATH --fc X Y` as CLI args, produce a single face PNG, and return it. This decouples expression-sheet generation from the drawing library and makes it easy for any generator to get a properly-drawn Luma face without importing the full curves module.
**Benefits:** Speeds up Maya's expression sheet iteration (any new expression renders immediately, no per-sheet rewiring); gives Jordan Reed a drop-in replace for _draw_luma_face_sprint() in SF02 — resolving the last remaining bespoke face code path and enabling the bezier face to be used in all Luma contexts. Also gives critics a quick tool to preview any expression delta combination without regenerating the full sheet.
