**Idea:** Build a static draw-order linter (`LTG_TOOL_draw_order_lint_v001.py`) that parses generator `.py` files and flags potential out-of-order draw calls. The linter would use simple pattern matching to detect when "head" or "face" draw calls appear before "body" calls, when "outline" draw calls appear before "fill" calls, or when "shadow" draw calls appear after the element they should darken. It could also detect when `draw = ImageDraw.Draw(img)` is NOT refreshed after `img.paste()` or `alpha_composite()`. This was one of the most common bugs in C22 (gaussian_glow dead alpha from missing refresh). The linter would not require executing the code — it scans for known dangerous patterns and outputs a per-file report. Could be integrated into the QA pipeline or run as a pre-commit check.
**Benefits:** Saves Maya Santos and Rin Yamamoto hours of debugging subtle layering bugs. Catches the "shadow after fill" class of errors before any image is generated. Makes it easier to onboard new generators without introducing paint-order regressions. Kai can build this in one cycle.
