**Idea:** Add a post-generation warmth lint hook to Miri character generators. The C38 audit found that CHAR-M-11 was corrected in master_palette.md in Cycle 32, but the error persisted in four active generator scripts for 6 more cycles before being caught by this audit. A lint hook called at the end of each Miri generator run would automatically check all CHAR-M values used in that generator against master_palette.md and warn if any are mismatched. This would catch drift immediately at generation time rather than requiring a manual audit cycles later. Implementation: add a `validate_miri_palette(script_path)` call in each generator's `if __name__ == "__main__":` block, backed by `LTG_TOOL_palette_warmth_lint`. Could also be integrated into `LTG_TOOL_ci_suite.py` as a new section alongside char_spec_lint.
**Benefits:** Eliminates the "palette fix in master_palette.md doesn't reach the generator" failure class. Catches CHAR-M-11 style errors at generation time (saves 6+ cycles of wrong output). Benefits Maya Santos (she generates most Miri expressions), Jordan Reed (generates kitchen/env scenes with Miri), and anyone doing a critique pass who currently needs to manually check slipper colors.
