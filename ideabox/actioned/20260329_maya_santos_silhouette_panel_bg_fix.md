**Idea:** The expression silhouette tool (LTG_TOOL_expression_silhouette_v002.py) consistently fails for all human characters regardless of pose improvements. Root cause identified this cycle: the IoM (Intersection over Minimum) metric is fundamentally biased toward high similarity for standing human characters because the shared body trunk area (torso, legs, head) dominates the silhouette comparison. Even dramatically different arm poses (head-grab bracket vs wide horizontal startle) score 96.7% similar. Two specific bugs compounding this: (1) Panel backgrounds with blue values close to character shirt blue (STRIPE_A=184 in B channel, BG panels with B≈200) may fall within BG_TOLERANCE=45, causing shirt pixels to be misclassified as background; (2) Dark label bars inside panel bounds corrupt corner sampling. Proposed fix: add a --metric option to the tool that switches to CONTOUR comparison (comparing only the outer edge/outline of the silhouette) rather than filled area comparison. Contour metrics would detect arm position changes even when the body trunk is shared. Alternatively, add a --body-subtract mode that: (a) computes the "base" silhouette from the smallest expression (most contracted body), (b) subtracts that base from all other silhouettes, and (c) compares only the DELTA pixels (the parts that change between expressions). This delta-mode would directly measure arm differentiation independent of the shared body.
**Benefits:** Would finally give Maya (and Alex) a tool that can actually measure silhouette differentiation for human characters. Currently the tool cannot distinguish dramatically different poses from nearly identical ones, making it misleading. All human character sheets have been producing FAIL results for 3+ cycles without valid diagnostic signal.
