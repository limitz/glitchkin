**Idea:** Add an automated silhouette-differentiation test to the QA pipeline for expression sheets. The test would extract each panel from an expression sheet PNG, flood-fill the character region to black, and compare the resulting silhouettes against each other via pixel overlap ratio. If any two panels have an overlap ratio above a threshold (e.g., 0.85 = 85% identical silhouette), the test flags them as "insufficiently differentiated" and returns FAIL. This would have caught the generic-expression problem in Luma v001–v007 automatically before critique, rather than waiting for Nkechi to flag it by eye. The tool would live in output/tools/ alongside LTG_TOOL_char_diff_v001.py. It could be run on every new expression sheet before check-in.
**Benefits:** Critics (especially Nkechi Adeyemi and Daisuke Matsumoto) consistently flag undifferentiated expressions as a recurring issue. Automating this check removes the dependency on a critique cycle to catch it. Maya and any future character designer would get instant feedback on whether their expression silhouettes are truly distinct. Tool would be reusable for any character's expression sheet.
