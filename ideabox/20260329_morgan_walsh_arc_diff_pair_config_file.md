**Idea:** Make the `ARC_DIFF_PAIRS` configuration in `LTG_TOOL_precritique_qa.py` externally configurable via a JSON file (`arc_diff_config.json` in `output/tools/`). Currently, the pairs are hardcoded in the Python constant at the top of the file. As the project grows — more acts, more contact sheet versions per cycle — the hardcoded list will need manual updates every cycle and will cause merge conflicts when multiple team members touch the tool. A JSON config file would let Lee Tanaka (or the storyboard team) register new pairs without touching pipeline Python. Format: `{"pairs": [{"label": "...", "old": "relative/path.png", "new": "relative/path.png", "diff_output": "production/filename.png"}]}`. The precritique_qa tool would load this at startup, falling back to the hardcoded `ARC_DIFF_PAIRS` constant if the file is absent (backwards compatible). Any pair where a file is missing is silently skipped (already the current behaviour).
**Benefits:** Reduces Morgan's workload each cycle (no Python edits for config changes), lets storyboard team self-service their contact sheet pair registrations, eliminates merge conflicts on the pipeline tool for config-only changes.
