**Idea:** The annotation_occupancy check uses a legacy-broad background classification on dark sheets (Byte, Glitch), producing WARN on panels that are intentionally near-void. Both Byte and Glitch sheets have this pre-existing WARN with 1–2% occupancy readings. The fix already applied to light sheets (precise annotation_bg_color matching) should be extended to dark sheets: detect if the panel background mean brightness is below a threshold (e.g., mean < 30), and if so, invert the classification logic — count bright pixels (>40 brightness) as annotation content directly, rather than subtracting background. This would correctly show Glitch/Byte amber/cyan annotations as high-occupancy content. The calibrate tool (sheet_geometry_calibrate.py) already detects dark sheets via mean brightness — this same heuristic should drive the lint path selection.
**Benefits:** Eliminates the persistent annotation_occupancy false WARN on Byte and Glitch motion sheets. Currently 2 of the 5 motion sheets (Byte + Glitch) always WARN on this check. Fixing this would bring the motion lint baseline from PASS=10 WARN=2 to PASS=12 WARN=0 across all sheets — a clean all-PASS baseline supporting C46 critique cycle.
