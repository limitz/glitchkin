## P1 — char_spec_lint Miri Filename Pattern

`LTG_TOOL_char_spec_lint.py` line 810 looks for:
```
LTG_TOOL_grandma_miri_expression_sheet_v*.py
```

The canonical file on disk is:
```
LTG_TOOL_grandma_miri_expression_sheet.py
```

No version suffix. The glob `_v*.py` never matches it — so M004 will always fail to find the Miri generator.

Fix: update the filename pattern in `char_spec_lint.py` to match the actual file name (no `_v*` suffix). Confirm M004 passes after the fix. Include a comment noting the no-version-suffix convention for this file.
