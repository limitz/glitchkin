## C54 — char_modular_lint False Positive Fix — COMPLETE

`LTG_TOOL_char_interface.py` upgraded to v1.1.0. CI suite version bumped to v2.2.0.

### Root Causes Found

Two false positive sources in the old scanner:

1. **Substring matching** — `draw_eye` matched `draw_eye_glow`; `draw_foot` matched `draw_footer`.
   Scene generator files `LTG_TOOL_sf_covetous_glitch.py` and `LTG_TOOL_sf_covetous_glitch_c43.py` were flagged for `draw_eye_glow` and `draw_footer`.

2. **Delegate wrappers counted as inline** — all 75 C53 baseline hits were scene generators that:
   - Import canonical char renderers (`from LTG_TOOL_char_luma import draw_luma` or with `as _draw_luma_canonical` alias)
   - Define thin wrapper functions (`def draw_luma_sitting(...)`, `def draw_byte_hovering(...)`, `def draw_glitch_large(...)`) that call the canonical module
   - Contain NO raw Cairo ctx.* or PIL draw.* primitives
   - These are the correct modular architecture pattern — they should NOT be flagged

### Fixes Applied

**Check 1 (body-part patterns):** Word-boundary regex `\bdef\s+draw_eye\b` — `draw_eye_glow` no longer matches.

**Check 2 (per-character functions):** Function body analysis:
- Extract function body lines after def
- Collect file-level imports from `LTG_TOOL_char_*` modules (handles both direct import and `as` aliases)
- `_func_body_is_delegate()`: returns True (not flagged) when:
  - No Cairo `ctx.*` primitives (move_to, curve_to, arc, fill, stroke, etc.)
  - No PIL `draw.*` primitives (polygon, ellipse, line, rectangle, etc.)
  - AND calls a name imported from a canonical char module OR references `_draw_*_canonical`

**Additional fix:** `draw_glitch(?!kin)` negative lookahead prevents `draw_glitchkin_swarm` matching the glitch branch.

### Expected Outcome

C53 baseline of 75 inline char draws → expected to drop to ~0 with the new scanner.
True inline character drawing (raw primitives without canonical delegation) will still be flagged correctly.
The check remains advisory WARN, not FAIL.
