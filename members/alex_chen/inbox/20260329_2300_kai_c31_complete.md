## Task 1 — Draw-Order Linter (DONE)

Built `output/tools/LTG_TOOL_draw_order_lint_v001.py`.

Static analysis tool (regex pattern matching, no AST, no execution). Detects four violation types:
- **W001** — HEAD/FACE draw call before BODY draw call
- **W002** — OUTLINE draw call before FILL for same element
- **W003** — SHADOW draw call after the element it should darken

**C31 run against all 114 LTG_TOOL_*.py files:**
- 59 PASS / 55 WARN / 0 ERROR
- Full report saved: `output/tools/LTG_TOOL_draw_order_lint_v001_report.txt`

**Key findings:**
- **W001 (head before body): 0 violations** — confirmed by C30 manual audit
- **W003 (shadow after element): 0 violations** — shadow discipline is good
- **W002 (outline before fill): false positives** — `draw.rectangle([...], fill=X, outline=Y)` is a single PIL call (valid), not a draw-order issue. Linter flags these conservatively. No real W002 violations found.
- **W004 (missing draw refresh): dominant warning** — affects 55 files, mostly older background generators (bg_grandma_kitchen, bg_tech_den, bg_other_side, style_frame_01, styleframe_discovery). These generators use `img_rgba = Image.alpha_composite(...)` and then continue drawing without refreshing the draw handle. Real risk in any generator that draws on `img` after compositing onto `img_rgba` or vice versa.

**Recommendation:** The W004 findings are a real pipeline risk. A team sprint to add draw refresh lines after all composite calls in active generators would be worthwhile — especially in styleframe_discovery_v004 (4 W004s) and bg_grandma_kitchen_v003 (8 W004s).
