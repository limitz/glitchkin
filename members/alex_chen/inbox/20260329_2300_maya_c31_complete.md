## Deliverable

**Tool:** `output/tools/LTG_TOOL_char_diff_v001.py`

Given two character sheet PNGs (reference + candidate), this tool pixel-samples
to estimate:
- Head height (px) — topmost dense non-background vertical region
- Total figure height (px) — full non-background vertical span
- Head-to-body ratio (figure_height / head_height)
- Eye width (px) — widest dark horizontal run in the head zone, capped at 35% image width to filter out structural lines
- Eye-to-head ratio (eye_width / head_height)

Outputs a JSON report + human-readable PASS/WARN/FAIL table.
Thresholds: WARN = >10% deviation, FAIL = >20% deviation.
Exit codes: 0=PASS, 1=WARN, 2=FAIL (pipeline-friendly).

PIL only — no Claude vision API.

## Key Operational Notes

- **Best input:** Full-scale single-character PNGs (turnarounds, color models). Eye
  detection most reliable at these scales.
- **Expression sheets:** Use bbox to isolate one panel (e.g. `0 0 400 450` for panel 1
  of a 1200x900 3×2 sheet). Eye metrics unreliable at panel scale — focus on
  figure_height and head_body_ratio.
- **Canonical usage before critique:** Compare new sheet against v007 turnaround
  FRONT panel as reference. Any proportion drift >10% will appear as WARN.
