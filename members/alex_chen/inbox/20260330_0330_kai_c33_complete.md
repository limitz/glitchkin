## Task 1 — Stub Integrity Linter

**File:** `output/tools/LTG_TOOL_stub_linter_v001.py`

Scans all `.py` files in `output/tools/` for broken imports referencing deleted
originals (old-style `LTG_CHAR_*`, `LTG_COLOR_*`, `LTG_BRAND_*`, `LTG_ENV_*` prefixes).

For each broken import reports:
- Which file contains it and exact line
- Whether the target `.py` exists on disk
- Whether a canonical `LTG_TOOL_*` replacement exists

Status levels: PASS / WARN (target on disk) / ERROR (target missing).

**Key feature:** `--pre-commit` mode exits code 1 on any ERROR — ready for use as
a pre-commit gate to prevent commits that would break stub chains.

API: `lint_file()`, `lint_directory()`, `format_report()`

## Task 2 — Glitch Spec Linter

**File:** `output/tools/LTG_TOOL_glitch_spec_lint_v001.py`

Validates Glitchkin generator files against `output/characters/main/glitch.md`.
Only lints files that actually draw Glitch (contain `diamond_pts`/`CORRUPT_AMB`/
`draw_glitch_body` markers). Non-Glitch files are SKIP (not counted).

**8 checks implemented:**
- G001: rx in [28–52], ry in [24–46] (spec ref: rx=38, ry=34)
- G002: ry > rx (diamond must be taller than wide)
- G003: Multi-panel files have ≥2 unique expression names
- G004: HOT_MAG crack drawn AFTER body fill (stacking order spec §2.3)
- G005: UV_PURPLE shadow offset (+3,+4) present (spec §2.2)
- G006: No organic/skin-tone RGB tuples in fill parameters
- G007: VOID_BLACK polygon outline present
- G008: Interior states (YEARNING/COVETOUS/HOLLOW) use bilateral eyes (spec §6.3)

API: `lint_file()`, `lint_directory()`, `format_report()`
