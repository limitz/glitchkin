## Task 1 — Warm/Cool Separation Root Cause Analysis

Brief written: `output/production/warm_cool_analysis_c35.md`

**Finding: The metric design is wrong for our palette, not the frames.**

The `_check_warm_cool()` metric splits each image into top/bottom halves and compares median
hue. It was designed for naturalistic lighting (warm ground, cool sky). Our three-world
palette applies temperature as a uniform frame-wide world signal — not a vertical gradient.

- SF01 (warm, 25.2° separation): Both halves are warm. Metric detects a warm-warm split,
  not a warm/cool contrast. Frame is intentionally all-warm. No correction needed.
- SF02 (contested, 9.2°): Both halves read as cool storm sky. Warm window accents are too
  few pixels to shift median. Storm dominates by design. No correction needed.
- SF03 (cold, 4.3°): Maximum cold uniformity throughout. No warm light sources. Correct.
- SF04 (warm, 1.5°): Full-frame lamp-lit domestic room. CRT Byte Teal pixels present
  (n=17,961) but can't shift the median against the warm room. No correction needed.

**Recommendation:** Kai Nakamura should extend the QA tool with an `expected_temp` parameter
("warm"/"cool"/"contested") so frames are checked against their intended temperature, not a
vertical split. Ideabox idea filed: `20260329_sam_kowalski_warm_cool_by_intent.md`.

## Task 2 — SUNLIT_AMBER Drift in SF04 (Nkechi C14 — Δ15.7°)

**Finding: Tool false positive. Generator is correct by design.**

`SUNLIT_AMBER = (212,146,58)` is defined in `LTG_TOOL_styleframe_luma_byte_v004.py` at
canonical value (RW-03, hue 34.3°) but is **never placed in any draw call** — it is an
unused constant. The lamp halo uses `(255,200,80)` at hue 41.1° — an incandescent lamp
amber, intentionally warmer/yellower than outdoor sunlit RW-03.

The Δ15.7° from the QA tool is caused by the tool sampling skin pixels (Luma's warm-lit
skin, hue ~18°) as its nearest warm hit. This is the same skin-tone false positive class
as C26+.

The warm character of SF04 is correct: lamp amber (41°) for indoor lamp glow is right;
SUNLIT_AMBER (34°) is for sunlit-window/exterior surfaces. No visual change needed.

Added comment to line 69 of the SF04 generator documenting this distinction.
