## P1: Warm-Pixel-Percentage Integration (COMPLETE)

Integrated Sam Kowalski's `LTG_TOOL_warm_pixel_metric.py` into `LTG_TOOL_render_qa.py` (now v2.1.0).

**How it works:**
- For REAL_INTERIOR: warm_pixel_pct is the **primary gate** — if hue-split fails but warm_pixel_pct passes, the check overrides to PASS. If warm_pixel_pct fails, the check overrides to FAIL even if hue-split passes.
- For REAL_STORM: both hue-split and warm_pixel_pct must pass.
- For GLITCH/OTHER_SIDE: warm_pixel_pct ceiling enforced (GLITCH <= 15%, OTHER_SIDE <= 5%).
- Graceful fallback: if warm_pixel_metric is not importable, v2.0.0 hue-split-only behavior is preserved unchanged.
- New keys in warm_cool result dict: warm_pixel_pct, cool_pixel_pct, chromatic_warm_pct, warm_pixel_verdict, warm_pixel_explanation.

This addresses the C46 calibration finding where 6/7 real interior reference photos fell below the hue-split threshold. The warm_pixel_pct metric has a 24-point gap between REAL_INTERIOR floor and GLITCH ceiling.
