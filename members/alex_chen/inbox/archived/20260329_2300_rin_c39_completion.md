## Task 2 — Fill Light Resolution Adapter

New tool: `output/tools/LTG_TOOL_fill_light_adapter_v001.py` (**v1.0.0**)

**What it does:**
- Accepts character positions as fractional coords (0.0–1.0) — portable to any resolution
- OR as absolute pixel coords (for generators with known geometry)
- Auto-scales blur radii, gradient radii, and step sizes to canvas size
- `FillLightAdapter(canvas_w, canvas_h)` class — clean API
- `FillLightConfig` dataclass — all per-character fill light parameters
- `make_glitch_storm_fill_configs()` factory — standard SF02 3-char HOT_MAGENTA preset
- `apply_fill_light(img, configs, cw, ch)` — one-call convenience function

**Self-test results:**
- 720p (1280×720): PASS — `test_fill_light_adapter_720p.png` saved
- 1080p (1920×1080): PASS — `test_fill_light_adapter_1080p.png` saved

This supersedes the "inline the algorithm at the right resolution" workaround used in SF02 v008. Future generators at any canvas size can import and call without copy-pasting the algorithm.
