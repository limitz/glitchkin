## Fix 1: spec_sync_ci G002 False Positive

**Root cause identified and fixed.**

The `LTG_TOOL_glitch_spec_lint_v001.py` file's own docstring contained:
```
rx must be in [30, 50] (spec reference: rx=38)
ry must be in [28, 46] (spec reference: ry=34)
```
The `_RX_ASSIGN` and `_RY_ASSIGN` regexes (`\brx\s*=\s*(\d+)`) matched these docstring reference values. Since `ry=34 <= rx=38`, this triggered a false G002 FAIL.

**Fix:** Added a G002 suppression entry for `LTG_TOOL_glitch_spec_lint_v001.py` to `glitch_spec_suppressions.json` with reason documented. The suppression list already loads correctly in `lint_directory()` (v1.2.0). The CI suite now reproduces the C37 report correctly.

**Note:** I also submitted an ideabox idea proposing docstring-stripping as a longer-term fix to prevent this entire class of false positive across all regex linters.
