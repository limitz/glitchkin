## Bug 1 — 3/4 view: legs look like side view

**Root cause:** `_draw_luma_threequarter()` sets `near_leg_x = hip_cx` and `far_leg_x = hip_cx` — both legs at the same x. That's side-view leg logic. In a 3/4 view you can see lateral spread.

**Fix:** Give the 3/4 legs a lateral spread, reduced from front view (roughly 60–65% of front's `leg_offset`). Use the same `fl_x` / `fr_x` pattern as `_draw_luma_front()` but with a narrower offset:

```
leg_offset_3q = head_r * 0.40 * 0.62 * spec.get("stance_wide", 1.0)
fl_x = hip_cx - leg_offset_3q   # character's left leg
fr_x = hip_cx + leg_offset_3q   # character's right leg
```

For right-facing 3/4, the near leg (character's right = `fr_x`) should be slightly forward — lower foot Y (closer to ground), more prominent. Far leg (character's left = `fl_x`) gets a small foot lift (+0.04*head_r) to read as receding. Draw far leg first (behind), then near leg on top.

Check `spec["weight_front"]` to confirm which leg takes the weight and which lifts — use that to assign front/back_foot_lift correctly.

---

## Bug 2 — Side views: legs not centered

**Root cause:** Legs are set to `near_leg_x = far_leg_x = hip_cx`. But `hip_cx = cx + hip_shift`, and the expression spec's `hip_shift` is non-zero (CURIOUS has a hip sway). So legs land offset from canvas center.

**Fix:** In the side view (both `_draw_luma_on_context` and `_draw_luma_side_l`), anchor the legs to `cx`, not `hip_cx`. The hip sway affects the waist and torso, but the feet stay planted at center:

```python
near_leg_x = cx
far_leg_x  = cx
```

The leg bezier top still connects up to `torso_bot_y` at `hip_cx` — that's fine, the slight diagonal in the upper leg from `hip_cx` to `cx` reads naturally as a weight-shifted leg. Only the foot/ankle should land at `cx`.

Apply the same fix to `_draw_luma_side_l`.
