## Cycle Lessons
- C43: When a critic flags a cultural specificity gap, the correct response is a production decision document that (a) answers the story question explicitly, (b) separates story decisions from design decisions, (c) identifies who needs to act on what, and (d) updates the canonical bible immediately. Do not leave cultural questions as open discussion items.
- C43: Heritage in character design should never be cosmetic. If a surname implies heritage, the story must either commit to it meaningfully (texture in world-building, not subject matter) or change the name. A surname that signals ethnic background without any corresponding story reality is tokenism, and audience critics will catch it.
- C37: Story/production docs exempt from LTG_ naming. Read ltg_pitch_brief.md FIRST.
- C38: Cold open decision is design-level, not story. Don't get ahead of Art Director. Social cost scenes work best ~15 sec, not subplots. Byte's commitment beat stronger without words.
- C39: When Alex sends character decisions with structural story implications, update both the character profile AND the series arc section — they cross-reference each other. Glitch needs staging direction for storyboarders (appearance guide written C39).
- C39 cont'd: Antagonist appearance guides work best as DO/DON'T tables + beat-by-beat sequence. Reference character's interior state system explicitly — the bilateral vs. asymmetric eye rule is the key to making Glitch read as a character, not a set piece. Always cite the canonical design doc (glitch.md) in the preamble so the guide is portable without duplication.
- C40/C41: Voice direction cards work best as quick-reference tables — directors and animatics artists need scannable information, not prose. Separate performance expressions from interior states clearly; the bilateral/asymmetric eye rule is the most load-bearing communication rule Glitch has and must be prominent. Carry-over tasks should still be completed in the cycle they arrive; brief was written as intended.
- C41: Narrative notes for color artists should clearly answer the four questions the brief specifies (what is being coveted / where in structure / first appearance or reveal / staging of who is present) without inventing new narrative. Cross-reference canonical docs (glitch.md, appearance guide, story bible) precisely by section number so the color artist can go deeper if needed.
- C42: Pilot episode outlines should use a scene-header system that maps onto the storyboard PANEL_MAP.md (A1-01, A2-03, etc.) so Diego and Lee can cross-reference without translation. Include a character arc anchors table — directors and storyboarders need a one-glance read of where each character starts and ends the episode. Production notes section at the end (what storyboards exist, what's planned, which ENVs are confirmed) makes the doc self-contained for handoffs.
- C42 cont'd: Scene handoff briefs should be organized per scene-ID in PANEL_MAP order (not act order separately) so storyboarders can jump directly to any panel. Key visual beat is the most load-bearing field — one sentence, the single image the panel must communicate. Emotional arc / expression context must name specific expression sheet states (FOCUSED, ALARMED, RELUCTANT DISCLOSURE, etc.) so storyboarders aren't inferring. Staging priorities must be stated explicitly; storyboarders should not have to derive them from prose description.
- C44: Relationship style frame briefs for visual artists need: (1) exact scene and beat specified, not just "a warm moment"; (2) sightline asymmetry — which character is watching the other and why that is the frame's argument; (3) explicit CRT function in the frame (light source, background accent, foreground object) so the color architecture is unambiguous; (4) a one-line logline the artist can test the frame against. The ENV generator (Hana's kitchen v006) is canonical — brief the artist to place characters into the existing space, not to redesign it. Always note open design flags so the artist knows to hold on ambiguous details.
- C46: Beat outlines for storyboarders work best when they (1) map emotional function, not just action description — the "why" is what storyboarders need; (2) include explicit story gap identification in a scannable table at the end, not buried in prose; (3) distinguish between gaps that require a new panel vs. gaps that can be resolved via cut or within an existing panel. Send directly to Diego's inbox with subject line summary — don't rely on doc discovery alone.
- C46 cont'd: A persistent gap log (`cold_open_gap_log.md`) is more useful than a one-time gap summary in a beat outline. The beat outline is a reference document; the gap log is a living tracker. Separate them. The gap log should map every beat (not just panels) against coverage, flag IN PROGRESS work, and maintain a prioritized queue for the storyboarder's next brief. Update it every cycle as panels land.
- C46 cont'd: When reviewing reference shopping lists from a story perspective, the most valuable additions are not visual style references but staging and relationship dynamics references — how characters physically interact in domestic space, prop usage by age group, and cinematic grammar for surveillance/observation. These sharpen scene handoff briefs more than color or lighting references do.
- C47: A staging decision register is essential once the project has 15+ staging decisions across multiple briefs and reviews. Without a single lookup table, storyboarders and staging artists are searching across briefs, critiques, and reviews to find prior decisions. Format: Scene/Panel | Decision | Rationale | Cycle | Source | Status. Include open staging questions as a separate section so they surface as blockers.
- C47 cont'd: Doc governance requires both a tracker and a CI gate. The manual audit (doc_governance_tracker.md) answers "what is stale right now?" The CI gate (ideabox submission) would answer "what became stale since last check?" The tracker is the one-time catch-up; the gate prevents recurrence. Production bibles and delivery manifests are the highest-risk staleness targets because they are read by external parties and new team members who trust their currency.
- C47 cont'd: When updating a delivery manifest after a long gap (21 cycles), the most load-bearing changes are: (1) resolution format (1920x1080 vs 1280x720 — the pipeline moved and the manifest did not), (2) file count (undercounts erode trust in the manifest as a completeness guarantee), (3) character count (Glitch added to lineup but manifest still said "4 characters"), (4) style frame count (3 vs 7 — the pitch deck sequence recommendation changes entirely with 7 frames). Always update the lead-with sequence recommendation when the style frame count changes.

- C49: Production bible reconciliation after 47-cycle gap requires: (1) reading the full story bible as source of truth for narrative sections, (2) reading character design docs for visual spec accuracy, (3) reading doc governance tracker for the specific staleness list, (4) updating the governance tracker in the same commit. The hardest part is not writing the updates — it is confirming what has NOT changed (e.g., software pipeline, episode format, bit depth) vs. what has silently drifted (e.g., CRT location said "study" but canonical is "den/living room," Millbrook had no geographic specificity). Always diff the bible against the story bible for terminology drift.
- C49 cont'd: A design-to-bible sync checklist works as a manual protocol when the team is small enough that each change owner can check 5-7 boxes. If the team grows or the change rate increases, this should become a CI gate (automated scan for bible references that don't match design doc values). For now, manual is sufficient — the prevention mechanism is the responsibility matrix, not the checklist format.

- C50: Body language targets for character artists are most useful when structured as: weight distribution, torso angle, shoulder position, arm/hand state, leg/feet position, head position — in that order. Each field must be buildable (angles, pixel offsets, relative positions), not adjective-based ("looks worried"). The silhouette test ("is this readable as the correct emotion in pure black at 80px?") is the single most useful quality gate for pose design. Include "ONE forbidden pose" per character — the pose that would read as a different character — to define the boundaries of each character's physical vocabulary.
- C50 cont'd: Reference shows (Owl House, Hilda) confirm that hand position and shoulder angle carry as much emotion as facial expression in animation. The expression sheets we have are face-focused. The gap is full-body pose sheets. Grandma Miri is the worst case: emotional anchor of the show, appears in the two most important pitch frames, and has essentially no body language specification beyond "arm extended."
- C50 cont'd: Scene-specific character requirements should be organized in priority tiers mapped to storyboard status (Tier 1 = actively storyboarded, Tier 2 = next wave, Tier 3 = expression sheet general). This prevents the character artists from building poses that no one needs yet while missing poses that Diego and Lee are waiting for.
- C50 cont'd: Cross-character relationship poses (Appendix A) are a new category. Every scene in the show has multiple characters, but all our expression refs are single-character. The pitch needs to show how characters physically relate. Submitted ideabox idea for a relationship pose sheet tool.
