## Assignment: Bezier/Spline Curve Drawing Library

Our characters look like assembled geometry because every body part is drawn with `ImageDraw.rectangle()`, `ellipse()`, and straight-line `polygon()`. We need a curve library that lets us draw organic shapes.

### Requirements:

1. **`draw_bezier_path(draw, points, control_points, fill, outline, width)`** — Draw a filled closed bezier path on a PIL ImageDraw surface. Takes a list of anchor points and control points, draws a smooth curve through them.

2. **`draw_bezier_stroke(draw, points, control_points, width, color)`** — Draw an open bezier stroke (for outlines, wrinkle lines, etc.).

3. **`tapered_limb(draw, p1, p2, w1, w2, fill, outline, width)`** — Draw a limb segment (arm, leg) as a tapered bezier shape. `p1` is the wide end (joint near torso), `p2` is the narrow end (hand/foot). `w1` and `w2` are widths at each end.

4. **`curved_torso(draw, shoulders, waist, fill, outline, width)`** — Draw a torso shape defined by shoulder points and waist points, with curved sides.

5. **`gesture_spine(head_pos, foot_pos, curve_amount, curve_direction)`** — Generate a gesture line (S-curve or C-curve) that the body will be constructed around. Returns control points along the spine.

### Technical Notes:
- PIL does not have native bezier support. Use `numpy` to compute bezier curve points, then draw as filled polygons with enough point density that they appear smooth.
- A cubic bezier needs ~50 sample points per segment to look smooth at our render scale.
- All functions must work with the 2x render scale pipeline and follow `docs/pil-standards.md`.
- The library file should be `output/tools/LTG_TOOL_curve_draw.py`.
- Include a visual test that draws sample shapes and saves to `output/production/curve_library_test_c50.png`.
