# MASTER INTERACTIVE-OBJECT OUTLINE EXCEPTION LIST
### "Luma & the Glitchkin" | Background & Environment Design
**Artist:** Jordan Reed | **Date:** 2026-03-29 | **Version:** 1.0
**Applies to:** All background environments — Luma's House Interior, Millbrook Main Street, Millbrook Middle School, The Glitch Layer

---

## The Core Rule

Background elements in this show are split into two rendering categories:

> **Interactive objects** (character can or will touch/use/react to them) receive **thin outlines** to tie them to the character layer. This signals to the viewer: "this object is in play."
>
> **Atmospheric background elements** (present for environment, world-building, or visual storytelling — but not acted upon in a given scene) receive **NO outline** and use the pure painterly-flat treatment.

The same physical object can switch categories depending on the scene. A locker is a background element in a hallway establishing shot. It is an interactive object when Luma opens it. **The outline is a function of scene usage, not the object itself** — with the exceptions listed below, which are ALWAYS outlined or ALWAYS not.

---

## Outline Weight Specifications

| Outline Type | Pixel Width at 1920x1080 | Use Case |
|---|---|---|
| **Thin (interactive background props)** | **1.5 px** | Standard interactive object outline. Background props in the interactive state. |
| **Medium (featured interactive props)** | **2 px** | Props that are the explicit subject of a close-up shot or are being held/used by a character in the same frame. |
| **Heavy (character outline — reference only)** | **3 px** | Character outlines, listed here for reference and comparison only. Background outlines must NEVER exceed 2px. |
| **Hairline (structural edges only)** | **0.75 px** | Used ONLY for the slight depth edge of locker doors against locker frames, desk edges where needed for legibility, and similar structural micro-edges. Not an "outline" per se — a depth indicator. |

**Outline color:** Deep cocoa (#3B2820) for all backgrounds. This is the same as the character outline color. Using the same color keeps props connected to the character layer. Do NOT use black (#000000) — it reads too heavy against the painterly backgrounds.

## ALWAYS-OUTLINED ELEMENTS
*(Interactive regardless of scene — always receive thin 1.5px outline)*

These elements are established as recurring interactive objects. They receive outlines in every shot where they are in the midground or foreground, even if no character is actively using them in that shot. Their outline is a standing invitation — "this can be touched."

### Glitch Layer

| Element | Location | Reason |
|---|---|---|
| Any platform where a character stands or is about to stand | Wherever staged | The platform edge outline is both structural (clarifies the ground plane) and interactive. Only platforms in the character's staging zone — NOT every platform in the composition. |
| Code waterfall columns (when characters are near/in them) | Whenever within 2 platform widths of a character | Waterfalls become interactive when characters approach or pass through them. Outline the column edges only — not every character in the waterfall. |
| Corruption Zones (when in midground or closer) | Wherever staged | Danger objects. Their outline (in Hot Magenta, #FF2D6B — the ONLY exception to the deep cocoa outline color rule) signals active threat. |

**Corruption Zone outline exception:** Corruption Zones receive a **1.5px Hot Magenta (#FF2D6B) outline** rather than deep cocoa. This is the only background element with a non-cocoa outline color. The magenta outline extends the danger color vocabulary and ensures Corruption Zones are immediately readable as "different category of wrong."

## NEVER-OUTLINED ELEMENTS
*(Atmospheric regardless of scene — never receive outline)*

These elements must NEVER receive an outline, even in close-up shots, even when they are compositionally prominent. They are always part of the atmospheric layer. If they start getting outlines, the visual language of the show breaks.

### Universal (All Environments)

| Element | Reason |
|---|---|
| Walls, floors, ceilings | Architectural planes. They are the container, not the content. |
| Light shapes (pools, shafts, glows) | Light has no edge in this style. Light is achieved through color shape overlap and opacity — never an outlined shape. |
| Shadow shapes | Same reasoning as light. Shadows are flat color overlays, not outlined objects. |
| Texture overlays (grain, scan-line, dither) | These are rendering technique layers, not objects. Never outlined. |
| Distant background elements (z=3 layer or farther) | Background layer elements always use painterly-flat no-outline treatment. Even if an object is interactive in concept (a building you could enter), if it is in the background layer of this shot, it has no outline. |

## Ambiguous Cases — The Rule

When an element is not clearly listed above, apply this decision tree:

**Step 1: Is this element in the background layer (z=3) of the current shot?**
- YES → No outline. Background layer elements are always no-outline.
- NO → Continue to Step 2.

**Step 2: Will a character touch, use, open, read, pick up, or directly react to this specific object in this scene OR in the next 3 scenes?**
- YES → Thin outline (1.5px, deep cocoa #3B2820). Establish it as interactive in advance.
- NO → Continue to Step 3.

**Step 3: Is this object a recurring prop that has been previously established as interactive in an earlier scene?**
- YES → Thin outline. Once established as interactive, it retains its outline when in midground or foreground.
- NO → No outline. Painterly-flat treatment.

**If still ambiguous:** Default to NO outline. It is always easier to ADD an outline in a revision than to explain to a director why a background object was outlined for no reason.

---

## Transition Protocol — When Objects Change State

Some objects transition from no-outline (background) to outline (interactive) across episodes. The transition must be handled correctly:

- **Do not outline an object in Episode 1 if it first becomes interactive in Episode 4.** The outline is applied from the episode where the interaction begins (or 1-2 episodes before, to establish it, if that serves the storytelling).
- **Exception:** Objects on the Always-Outlined list are outlined from their first appearance, even if their interaction moment is later in the season. (E.g., the Cable Drawer Cabinet is outlined from Episode 1 even though it first becomes plot-relevant in Episode 4.)
- **The Glitch escalation props** (locker 147, classroom plant, fish tank): These follow a special protocol. They are NOT outlined until their Glitch activation begins. Before activation: no outline. After activation: thin outline. This makes the outline itself a subtle signal to the careful viewer that something has changed — even before the visual glitch effect is obvious.
