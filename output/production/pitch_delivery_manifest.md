## Delivery Notes

### File Format
- All PNGs should be delivered at full resolution (1920×1080 or 1280×720 as generated). Do not downscale.
- Markdown documents should be converted to PDF for buyer-facing delivery. Internal reference files can remain as MD.
- No generators, no Python tools, no source code included in the external package.

### What NOT to Include in External Delivery
- Any file with `_v001` through `_v004` where a `_v005` or later exists (always ship latest)
- `*_guides.png` files (construction guide variants — internal only)
- Individual storyboard panels (deliver contact sheets only unless requested)
- Any `statement_of_work_*` documents
- Any `critic_feedback_*` or `critique_*` documents
- Any `LTG_TOOL_*` scripts
- `byte.md`, `luma.md`, `cosmo.md` character spec files (internal production reference only)

### Pre-Ship Checklist
- [ ] All file paths resolve (no broken references)
- [ ] All PNGs render correctly at 100% scale
- [ ] Logo tagline has been removed from `show_logo.png` (placeholder "A cartoon series by the Dream Team" must not ship)
- [ ] `ltg_pitch_brief.md` converted to PDF
- [ ] File count confirmed against this manifest

### Outstanding Before External Delivery
1. **Logo check** — verify `show_logo.png` does not contain placeholder tagline text. If present, either use `LTG_BRAND_logo_asymmetric.png` as primary, or strip the tagline.
2. **Color model PNGs for Luma/Byte/Cosmo** — these do not yet have standalone visual PNG exports (only `.md` specs). Flag for a future cycle if the studio requests visual color reference sheets.

> **Note (Cycle 26):** The stylization post-processing pipeline has been retired by producer directive. All `*_styled*.png` files have been removed. Style frames ship as clean generated PNGs. Stylization tools (`LTG_TOOL_stylize_handdrawn.py`, `v002.py`, `LTG_TOOL_batch_stylize.py`) moved to `output/tools/legacy/`.
